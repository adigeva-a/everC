ALTERNATIVES ANALYSIS

Three credible paths. One recommendation.

REJECTED

Better Detection Precision

Even 100% precision still generates alerts. Every true positive still requires manual investigation. Improves signal quality — does not touch operational scalability.

COMPLEMENTARY

Risk-Based Monitoring Tiers

Instead of monitoring all merchants equally, segment the portfolio into dynamic risk tiers: High risk → continuous deep monitoring, all violations surfaced; Medium risk → monthly monitoring, violations batched weekly; Low risk → quarterly monitoring, only significant violations surfaced. If 60–70% of merchants are genuinely low risk — which customers already assume — this dramatically reduces violation volume at the source.

WHY IT'S STRONG:
  • Directly reduces alert volume at the source
  • No workflow or behavior change required
  • No adoption friction
  • Fully respects the internal data constraint
  • Can be deployed quickly as a monitoring strategy shift
It also directly addresses demand suppression: A customer afraid to send 50,000 merchants now hears: "Your low-risk portfolio will generate minimal alerts." → They send more merchants → G2RS unlocks growth.
WHERE IT FALLS SHORT:
  • Reduces volume — but not investigation time. Each surfaced violation still requires the same manual investigation. Queue is smaller → but each item is equally expensive.
  • Introduces miscategorization risk. Incorrect tiering = missed violations for months → regulatory exposure → customer trust risk. Risk-averse customers may hesitate to adopt.
  • Does not create a learning system. Tiering is largely static. No feedback loop. No compounding intelligence over time.
  • Does not evolve the workflow. API and UI customers experience no change in investigation process. Only fewer alerts — not better handling.
CHOSEN ✓

Merchant Risk Co-Pilot

Attacks the root cause: structural mismatch between how alerts arrive and how investigations run. Solves investigation throughput, workflow inefficiency, context reconstruction, and cognitive switching. Compounds in value via learning loop.

STRATEGIC INSIGHT

These are not competing solutions.

They are complementary layers in the same system.

Together: fewer alerts × faster investigation per alert = full-system scalability.

Recommended Approach:

  1. Launch Co-Pilot first → prove investigation efficiency gains → establish baseline metrics
  2. Introduce risk-based monitoring tiers as a second layer → reduce incoming volume → amplify Co-Pilot impact

This is a systems-level solution, not a feature.

THE SOLUTION

A new operating model for investigations.

Not a feature list. A workflow redesign.

A.

Change the unit of work

Raw alerts → structured merchant cases grouped by violation type. The analyst works one investigation logic across multiple merchants.

B.

Pre-load context

Evidence pack auto-assembled on case open: Compass Score, EverC risk graph, merchant history. No manual lookup across 3+ tools.

C.

Constrain the decision

Targeted verification checklist: exactly what internal data to verify for this violation type. No guessing. Just execute.

D.

Accelerate understanding

AI case summary on demand. Plain-language context in 20 seconds. On-click only — prevents anchoring bias.
